This guide walks you through getting a running version of the app with your own bracket matchups.

## Prerequisites

Before you start, make sure you have:

- **Node.js** 18 or later
- **pnpm** installed globally (`npm i -g pnpm`)
- A **GitHub** account
- A **Vercel** account (free tier is sufficient)

## Steps

<Steps>
  <Step title="Fork the repository">
    Go to [github.com/bb220/madness-tracker](https://github.com/bb220/madness-tracker) and click **Fork** to create a copy under your own GitHub account.

    <Tip>
      Keeping the repo public makes the Vercel integration simpler on a free plan.
    </Tip>
  </Step>

  <Step title="Clone and install dependencies">
    Clone your forked repository and install dependencies with pnpm.

    <CodeGroup>

    ```bash Clone your fork
    git clone https://github.com/YOUR_USERNAME/madness-tracker.git
    cd madness-tracker
    ```

    ```bash Install dependencies
    pnpm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Customize your matchups">
    Open `app/page.tsx` and replace the `matchups` array with your own bracket data.

    The `Matchup` interface (defined in `components/matchup-card.tsx`) describes the shape of each entry:

    ```typescript
    export interface Matchup {
      myTeam: string      // Your team's name
      mySeed: number      // Your team's tournament seed
      myScore: number     // Your team's current score (0 if not started)
      theirTeam: string   // Opponent team's name
      theirSeed: number   // Opponent team's seed
      theirScore: number  // Opponent team's current score
      spread: string      // Point spread, e.g. "+8.5", "-2.5", or "TBD"
      role: "dog" | "fav" | "tbd"  // Your role: underdog, favorite, or pending
      tipTime: string     // Tip-off time, e.g. "9:45 PM ET"
      network: string     // Broadcast network, e.g. "CBS", "TBS/truTV"
      date: string        // Game date, e.g. "Thu Mar 26"
      region: string      // Tournament region, e.g. "West", "South"
      opponent: string    // The bracket pool opponent you're facing
    }
    ```

    Here is an example matchup pulled directly from the source:

    ```typescript
    const matchups: Matchup[] = [
      {
        myTeam: "Arkansas",
        mySeed: 4,
        myScore: 88,
        theirTeam: "Arizona",
        theirSeed: 1,
        theirScore: 109,
        spread: "+8.5",
        role: "dog",
        tipTime: "9:45 PM ET",
        network: "CBS",
        date: "Thu Mar 26",
        region: "West",
        opponent: "Brady",
      },
    ]
    ```

    <Tip>
      Set `myScore` and `theirScore` to `0` for games that haven't started yet. The card will display a dash (`-`) instead of zero.
    </Tip>

    <Note>
      The `role` field controls the card's color accent and badge label. Use `"dog"` for underdog (blue), `"fav"` for favorite (green), and `"tbd"` when the spread isn't set yet (gray).
    </Note>
  </Step>

  <Step title="Run the development server">
    Start the local dev server to preview your changes.

    ```bash
    pnpm dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser. You should see your matchup cards rendered on the page.
  </Step>

  <Step title="Deploy to Vercel">
    Push your changes to your forked GitHub repository, then connect it to Vercel.

    ```bash
    git add app/page.tsx
    git commit -m "customize matchups"
    git push
    ```

    1. Go to [vercel.com](https://vercel.com) and click **Add New Project**.
    2. Import your forked `madness-tracker` repository.
    3. Leave all build settings at their defaults — Vercel detects Next.js automatically.
    4. Click **Deploy**.

    Once deployed, Vercel gives you a public URL. Every subsequent push to your repository (including automated pushes from Codex score-sync automations) will trigger a new deployment automatically.

    <Tip>
      Copy your Vercel deployment URL and bookmark it on your phone. The single-column layout is designed for mobile viewing.
    </Tip>
  </Step>
</Steps>
