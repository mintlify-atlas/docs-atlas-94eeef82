Brandon's Matchups is deployed on [Vercel](https://vercel.com), which provides automatic deployments on every push to the GitHub repository. Once configured, no manual deploys are ever needed.

## Vercel Deployment

<Steps>
  <Step title="Connect the GitHub repository">
    In the [Vercel dashboard](https://vercel.com/new), import the `madness-tracker` repository from GitHub. Vercel will request repository access during this step.
  </Step>

  <Step title="Let Vercel auto-detect the framework">
    Vercel automatically detects Next.js and configures the build settings:

    | Setting | Value |
    |---------|-------|
    | Framework | Next.js |
    | Build command | `pnpm build` |
    | Output directory | `.next` |
    | Install command | `pnpm install` |

    No manual configuration is required.
  </Step>

  <Step title="Deploy">
    Click **Deploy**. Vercel runs the initial build and publishes the site to a `.vercel.app` URL.
  </Step>

  <Step title="Push to trigger future deploys">
    Every subsequent push to the `main` branch triggers an automatic build and deployment. The live URL updates within approximately 30 seconds of a push landing on GitHub.
  </Step>
</Steps>

<Note>
  Once the initial Vercel project is connected, all future deploys are fully automatic. You never need to visit the Vercel dashboard again to publish score updates.
</Note>

## The Auto-Deploy Loop

Score updates flow from source to live site without any manual steps:

1. **Codex scrapes scores** — fetches current game scores from a live source
2. **Codex edits `app/page.tsx`** — updates `myScore` and `theirScore` fields in the matchups array
3. **Codex commits and pushes** — commits the change to the `main` branch on GitHub
4. **Vercel detects the push** — triggers an automatic build
5. **Live site updates** — the new deployment goes live within ~30 seconds

This loop runs continuously during tournament games. No human intervention is needed once the initial Vercel project is set up.

## Next.js Configuration

The `next.config.mjs` file contains two settings that keep deploys fast and friction-free:

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  typescript: {
    ignoreBuildErrors: true,
  },
  images: {
    unoptimized: true,
  },
}

export default nextConfig
```

### `typescript.ignoreBuildErrors`

Set to `true`. This allows Vercel to complete a production build even if there are minor TypeScript type errors. Since Codex edits `app/page.tsx` programmatically, this setting prevents a type warning from blocking a score update from going live.

### `images.unoptimized`

Set to `true`. Disables Next.js's built-in image optimization pipeline. The tracker does not use any `<Image>` components with remote sources, so optimization is unnecessary and disabling it simplifies the build.

## Local Development

Use `pnpm` to run the project locally.

```bash
# Start the local dev server (hot reload enabled)
pnpm dev
```

The dev server runs at `http://localhost:3000`. Changes to `app/page.tsx` (including score edits) reflect immediately without a page refresh.

```bash
# Build for production
pnpm build

# Run the production build locally
pnpm start
```

| Command | Purpose |
|---------|--------|
| `pnpm dev` | Local development at `http://localhost:3000` |
| `pnpm build` | Compiles the production Next.js bundle |
| `pnpm start` | Serves the compiled production build locally |
| `pnpm lint` | Runs ESLint across the project |
